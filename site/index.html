Testing infrastructure for Mesos frameworks

  • Test Driven Development for Mesos frameworks
  • Add a minimesos JUnit @ClassRule to start containerized Mesos cluster
  • Start the scheduler of your framework
  • Perform assertions on the cluster state via minimesos API
  • After the test is done the cluster is destroyed
  • Gradle
  • Maven

Step 1. Add the JitPack repository to your gradle build file:

repositories {
  // ...
  maven { url "https://jitpack.io" }
 }

Step 2. Add the dependency in the form:

dependencies {
  compile 'com.github.ContainerSolutions:minimesos:0.5.0'
 }

Step 3. Add a @ClassRule in your tests to start and destroy your Mesos test cluster

@ClassRule
 public static final MesosCluster CLUSTER = new MesosCluster(
  MesosClusterConfig.builder()
   .slaveResources(new String[] {"ports(*):[8080-8080]"} )
   .build()
 );

Step 1. Add the JitPack repository to your maven POM file:

<repository>
  <id>jitpack.io</id>
  <url>https://jitpack.io</url>
</repository>

Step 2. Add the dependency in the form:

<dependency>
  <groupId>com.containersolutions</groupId>
  <artifactId>minimesos</artifactId>
  <version>0.4.0</version>
</dependency>

Step 3. Add a @ClassRule in your tests to start and destroy your Mesos test cluster

@ClassRule
 public static final MesosCluster CLUSTER = new MesosCluster(
  MesosClusterConfig.builder()
   .slaveResources(new String[] {"ports(*):[8080-8080]"} )
   .build()
 );

SIMPLE

Start a Mesos cluster on the command line or via Java API.

ISOLATED

Mesos master, slave and Zookeeper processes run in separate Docker containers.

INTEGRATED

minimesos exposes framework, state and task information to its Cluster State API.
